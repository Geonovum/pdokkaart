Pdok.Api(config)
Creates an instance of the Pdok Api based on an optional config object more ...

If there is not a config object, all properties of the api have a default value.

The config object is a hash object like: 
{ loc:'150000,380000', div:'map' }

The OpenLayers.Util.getParameters() function returns such an object. Giving the possiblitiy to create an Api object which uses url params in just two lines of code:

   var config = OpenLayers.Util.getParameters();
   api = Pdok.Api(config);
All properties of the config object are overriding default values of Api
Parameters:
{Object} config
A hash object with all possible config key/value pairs
Field Detail
{String} bbox
A boundingbox (w,s,e,n) to zoom to. A commaseparated String of west,south,east,north
{Object} defaultPdokLayers
Object which holds a Map of an shortname/id to'layer'-configuration objects. All layer objects holds hold an layertype, and other(OpenLayers-option) properties specific for that type of (OpenLayers)-layer. Be carefull to use the right properties! Eg a OpenLayers-WMS-Layer has a property 'layers' (note the plural form), while the OpenLayers-WMTS-Layer has a property 'layer' (without s!)
